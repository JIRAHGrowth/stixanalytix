## TL;DR

Three wins. Three trade-offs. One null finding. **Net: directionally right, ceiling visible.**

**Wins:**
1. **PFC match goals: 14 hallucinated → 0** (gemini-2.5-pro old prompts → gemini-2.5-flash v3). The team-color self-audit + shooter-color requirement crushed the dominant-win goal-hallucination failure mode that triggered the May 26 dejection.
2. **PFC match saves: 55 → 7** (87% FP reduction on the worst-case lopsided match).
3. **Canary match (04-25) distribution recall: 31% → 81%** (Phase 2 baseline → v3). The model is finding far more real distributions when it's allowed to release the inflated `gk_short`/`gk_long` count.

**Trade-offs:**
1. **Goals over-corrected on PFC: 0% recall.** The evidence-count rule (Rule D, 2-of-3 affirmative) is now killing real goals on no-scoreboard youth video where the model can't reliably observe celebrations AND kickoffs AND scoreboard for the same event.
2. **Distribution precision still 2-10%** across the corpus. Rule E drops 18-35 dupes per match but the underlying over-tagging is structural.
3. **Save precision didn't lift meaningfully.** Rule F (antecedent-attack filter) **fired zero times in every match** — Gemini generates plausible-looking `preceding_attack` descriptions for hallucinated saves too. The rule design assumed empty/generic strings; the model writes detailed fiction.

**Null finding:**
- Static-cam sensor does NOT fix goal hallucinations on its own. Old prompts on the GK-cam still produced 15 phantom goals (all from celebrations bleeding into frame). Sensor changes can't bypass the model's training-distribution prior.

## Rule F null-firing — diagnosis

Rule F (saves antecedent-attack filter) dropped **0 events on every match**. Designed to catch saves where the model failed to describe an opposition attack. Inspection of bench output shows why:

The model fills `preceding_attack` with detailed, plausible-sounding descriptions even for hallucinated saves:

> "Opposition #9 received the ball at the top of the box on the left, took one touch to set, and struck low across goal toward the far post"

…where the event never happened. The model can't tell whether it's narrating a real event or confabulating one — and it writes equally fluent prose for both.

**The filter design assumption was wrong.** Empty/generic strings would have indicated the model couldn't justify the save. In practice, the model never has trouble justifying — it has a strong narrative prior even for invented events.

**This is the most important specific learning of this bench run.** Field-presence heuristics cannot detect hallucination once the model is fluent. The same applies to `evidence_shooter_color` on goals — the model can describe a "dark jersey with white sleeve striping" for a player it invented. Schema-required affirmative fields are catchable by inference, but their VALUES can't be sanity-checked from text alone.

**Implication:** the SFT path is the only path. Prompt + schema scaffolding shifts the failure mode but cannot eliminate it. The corrections → JSONL converter (committed `151aa32`) is the unblock.

## What I'd recommend

1. **Don't revert v3.** The PFC over-detection collapse and the dist recall lift on the canary are real, durable wins. The goal recall regression on matches that previously had 100% R is the cost — and on those matches the precision was 7-14%, so the "100% R" was over-reporting.
2. **Deploy v3 to Modal worker** to fill the (v3 × static GK-cam) cell of the 2×2. If v3 reduces 15 → <5 phantom goals on the GK-cam, the same-day dejection event was the worst-case input AND we have a meaningful response.
3. **Cancel work on stronger reconciliation rules.** Rule F's null-firing proves the ceiling. More rules will get the same outcome (`G` also dropped 0 events on every match).
4. **Pivot calendar time to SFT corpus growth.** The corrections converter is shipped (`151aa32`); now needs the 7 published-match videos copied into GCS so the JSONL build can complete. Then every new match Joshua reviews is +39 training rows automatically. The 500-row Vertex minimum is reachable in ~13 reviewed matches, the 200-match target is ~7,800 rows.
5. **Skip another round of prompt iteration.** Iterating further at this stage trades precision for recall (or vice versa) without moving the underlying ceiling. The next dollar of marginal improvement is in fine-tuning, not prompts.

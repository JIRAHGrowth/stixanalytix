Design exploration · 2026 · 02 Domain Primer mockup

Three takes on a StixAnalytix
training manual.

Each variant uses the same brand DNA — bone & emerald on warm near-black, Outfit display + DM Sans body, the rectangular goal-net as architectural motif. They diverge on editorial structure. Open each, decide which one we run with.

01
Editorial Classic
Single-column · Drop caps · Pull quotes

The most book-like of the three. Single 640px column centred on the page with generous margins. Drop-cap on the chapter opener. Italic pull-quotes set off a green hairline rule. The net motif appears as a horizontal strip section divider only — earning its impact by being rare.

Use this if: the docs will be read cover-to-cover. Maximum reading comfort over 30-45 min sittings. Closest to a Stripe Press playbook or coaching manual.

LayoutSingle column, 640px max
DensityLower — most whitespace
ToneAuthoritative, editorial, calm
RiskLess distinctly "us"

02
Brief with Marginalia
Two-column · Nav rail · Reference-dense

Narrow left rail on every inner page carries the section navigation, page number, big stat callouts, and inline asides. Wide right column for body copy. The body is two-column with sidebar quotes inline. Reference-manual energy — McKinsey brief / Anthropic engineering doc.

Use this if: labelers will scan-and-jump rather than read linearly. Best for the rubric & reference docs (03a, 03b). Also strongest for Nicolas' ML team — looks like a serious technical document.

Layout200px rail + main column
DensityHighest — most signal per page
ToneTechnical, navigable, dossier
RiskCan feel corporate over 30 pages

03
Magazine Spread
Feature article · Big display · Mixed-bg pages

Largest display type, most editorial drama. Cover treats each part as a feature article with a hero number + headline. Body pages use 4-column grids for cue lists. The diagram/data page goes half-dark / half-paper for visual rhythm. Big pull-quotes get full-width treatment with double rules.

Use this if: the docs need to feel premium and distinctly STIX. Best at hooking new readers (the cover sells the document). Less efficient if the reader is in scanning mode.

LayoutAsymmetric · 4-col body grids
DensityMedium — more breathing room
ToneCinematic, brand-forward, FT Weekend
RiskSlower to read; type can intimidate

How to compare

  1. Open all three in separate tabs. Look at the covers first — which one would you put in front of Nicolas?
  2. Now scroll to page 4 in each — the table page. Tables are the most-used pattern across all 7 docs. The one that handles tables best wins the most ground.
  3. Then scroll to page 5 — the diagram/data page. Will the keepers we work with respond to it?
  4. Pick one, OR mark up one variant's cover + another variant's table treatment — we can hybridize.
  5. Once picked, I build the master template that produces all 7 branded docs from the existing markdown.

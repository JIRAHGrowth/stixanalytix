01 · Why calibration

A rubric is a contract written in words.

Two careful labelers reading the same rubric will diverge on the gray areas. The rubric can't anticipate every edge case; only labeling does.

Calibration measures the divergence, surfaces the gray areas, and updates the rubric before the labelers scale to production. Without it, you don't have one consistent dataset — you have N datasets, one per labeler.

When to run it

  • Every time a new labeler joins.
  • Every time the rubric changes materially — new event type, new classification, changed definition.
  • Every quarter, as a drift check on existing labelers.

If you skip calibration, you don't have one dataset. You have N datasets, one per labeler — and the model can't learn from the contradiction.

First principle

The cohort

The cohort is the new labeler plus at least one experienced labeler. Three is better than two — disagreement between two people is hard to break ties on.

For the very first cohort (the one this onboarding package is built for), the experienced labeler is Joshua. New labelers calibrate against his rulings until at least one other labeler has been promoted past calibration; then he steps back from being the sole standard.

02 · The calibration match mix

Pick 5–8 matches that cover the variety.

Don't all-pick easy matches. Each characteristic stress-tests a different part of the rubric. Cover the spectrum or you'll only discover the failures after you've shipped.

Table 02.1 · Calibration set composition
Match characteristic Why it matters Count
Standard competitive (3-2 ish, both teams attack) Bread and butter. Many save events, normal distribution rates. 2
One-sided win for analyzed team (5-0 ish) Tests whether labelers correctly produce a LOW save count instead of inventing saves to fill a quota. 1–2
One-sided loss for analyzed team (0-5 ish) High save volume, many goals. Tests endurance and the rebound-asymmetry rule. 1
Low-quality video (phone, angled, no scoreboard) Tests honest use of Unclear and GK visible: No. 1
Match with a GK error leading to a goal Tests Missed/Misjudged classification. 1

The judah-vs-pfc-2026-05-23 match (15-0 win for analyzed team) is essential. It's the one Gemini hallucinated 51 of 55 saves on. If a labeler correctly produces 4 saves, they're calibrated. If they produce 30, they're chasing the model's output.

03 · The 4-round process

A learning loop, not a test.

The first calibration match almost never hits the IAA thresholds. That's the whole point — calibration is iterative.

00

Pre-reading

Before touching a match, the new labeler reads 01 Mission & Why, 02 Domain Primer, and 03a Rubric. Skim 03b Edge Case Log — they'll add to it during calibration.

01

Independent labels on match 1

Both labelers label the same match independently. No conferring. No looking at each other's spreadsheets. Time-box: 24 hours from receiving the match to submission. Submit as <match-name>__labeler-<initials>.json.

02

Agreement measurement

Run the comparison. Measure inter-annotator agreement on event recall, classification, timestamps, and Unclear usage. Thresholds on the next page.

03

Disagreement walkthrough

Working session, 1–2 hours per calibration match. Open the video together. For each disagreement: both labelers explain what they saw, the experienced labeler makes the canonical ruling, the ruling goes into 03b immediately. Edit the rubric in the same session if the disagreement exposes a gap.

04

Re-label match 1

The new labeler re-labels match 1 from scratch using the updated rubric. Goal: at least 90% agreement against the gold-standard version. If they hit it, move to the next calibration match. If they don't, identify which dimension they're failing and target the next match's coaching there.

05
Calibration
If they keep failing
Most labelers graduate within 3-5 matches. If they're still failing after 6+, the problem is the rubric, not the labeler. Fix the rubric and retest.

04 · IAA thresholds & graduation

Inter-annotator agreement — the production bar.

A labeler is promoted from "calibrating" to "production-ready" when, on a fresh calibration match they have never seen, they hit all four thresholds.

Table 04.1 · Production thresholds
DimensionWhat it measuresThreshold
Event recall Did both labelers find the same events? ≥ 90%
Event count by type Same number of goals / saves / distributions logged? within ±2
Classification agreement When both found the same event, did they classify it the same way? ≥ 85% on GK action, ≥ 90% on scoring team
Timestamp agreement When both found the same event, were timestamps within 5 seconds? ≥ 95% within ±5s
Unclear usage Is one labeler over-confident and the other under-confident? both at 10–25% on borderline classifications

Graduation criteria

On a fresh calibration match the labeler has never seen, all of:

  • Event recall ≥ 90%
  • Classification agreement on GK action ≥ 85%
  • Timestamp agreement ≥ 95% within ±5s
  • Zero false positives on the antecedent-attack rule (no save events with vague Play description)

05 · Graduation isn't unsupervised

The QA loop continues after graduation.

Graduated labelers are not unsupervised. Three ongoing checks catch drift before it corrupts the dataset.

Three ongoing checks

  • 5% spot check. Joshua (or another graduated labeler) re-labels a random 5% of every labeler's matches in shadow mode and compares.
  • Edge cases always go to the log. Even graduated labelers add new gray areas. The log is canonical; private rulings are forbidden.
  • Quarterly recalibration. Every labeler labels one fresh calibration match against the current gold standard. Catches drift from accumulated bad habits.

Pay model — base + accuracy bonus

The right model is base rate per match + accuracy bonus:

  • Base rate per match. Calibrated to roughly 1.5× the time a competent labeler takes (so an experienced labeler earns a premium).
  • Accuracy bonus. Paid against a sampled subset (5–10% of the labeler's matches) re-labeled by a graduated reviewer. Bonus scales with agreement against the gold standard.

Pay per match alone incentivizes speed over accuracy. Pay per hour alone incentivizes slowness. The split model rewards labelers who are both fast AND right.

For volunteers / equity helpers
Same QA loop applies but the accuracy bonus is replaced by recognition — your name in the dataset credits, project shoutouts, eventual reference letter if you stick.

The Edge Case Log is also signed — every ruling carries the labeler's name. That becomes a portfolio piece.

At ready-to-scale, you can give a calibrated labeler their first 5 production matches with confidence that what comes back is consistent training data — not personal interpretation.
